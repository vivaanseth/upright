# Posture v0.5.2

`v0.5.2` is a focused correctness release for the Posture beta line.

## Fixed

- Reminder timing now uses the main process's monotonic clock instead of accepting renderer-provided timestamps, preventing reminders from remaining suppressed because of cross-process clock differences.
- Camera streams and pose workers are cleaned up if either half of startup fails.
- In-progress calibration is cancelled when backing out, leaving calibration views, hiding setup before tracking starts, pausing, stopping, resetting, or quitting.
- The reminder sound setting now plays a subtle local-only cue when enabled.
- Adaptive sampling can recover toward higher FPS after sustained low latency and low dropped-frame rates.
- Media permission checks now require explicit video-only requests.
- Missing `app://` assets now return controlled 404 responses.

## Changed

- Runtime packages now live in production dependencies so production audits and release SBOM inputs include the real runtime surface.
- License auditing can fall back to installed package manifests when pnpm's license index is unavailable.
- Reminder nudges remain non-focus-stealing on display, then become focusable after user interaction.

## Notes

- Camera processing remains local to the device.
- No camera frames, images, videos, or raw landmark sequences are saved by Posture.
- This beta release still requires the documented compatibility, performance, and accuracy evidence before a stable compatibility claim.
