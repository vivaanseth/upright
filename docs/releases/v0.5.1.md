# Posture v0.5.1

`v0.5.1` is the first publishable beta release candidate for Posture's runtime, calibration, session, reminder, local-data, and release-readiness foundation.

It supersedes the failed `v0.5.0` tag workflow attempt. The `v0.5.0` app changes remain the beta baseline; `v0.5.1` fixes the Windows release-runner license audit so all native platform builders can complete.

## Highlights

- Camera lifecycle is tracked separately from posture quality.
- Calibration V2 is bound to the exact selected camera.
- Session summaries and exports use V2 schemas with crash recovery.
- Worker inference has bounded frame ownership, watchdog recovery, and adaptive sampling.
- Settings include calibration management, export, delete, reset, privacy, and attribution controls.
- CI includes package/privacy/license/audit checks in addition to formatting, linting, type checking, tests, builds, and Electron E2E.
- Windows release jobs can now run the license audit through the pnpm CLI path exposed by the runner.

## Install

These artifacts are unsigned beta builds.

- macOS: download the universal DMG or ZIP. If Gatekeeper warns, use Finder's Open action.
- Windows: download the x64 installer or portable ZIP. SmartScreen may warn because the build is unsigned.
- Linux: prefer AppImage for portable launch, DEB for Ubuntu/Debian, RPM for Fedora-family distributions, or tar.gz for a manual install.

## Known limitations

- The Codex Security deep scan was intentionally excluded from this beta completion pass.
- Automatic updates remain disabled until macOS and Windows signing are available.
- Native OS notifications remain disabled; Posture uses its own app nudge.
- Stable compatibility claims require the full manual matrix in `docs/qa/compatibility.md`.
