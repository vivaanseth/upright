# Posture v0.1.2 inference hotfix

This prerelease replaces the unpublished `v0.1.1` build. It repairs the bundled MediaPipe runtime so camera frames reach local pose inference during onboarding and calibration. It remains unsigned early software and is not yet represented as stable or fully validated across the complete support matrix.

## Fixed

- Restore all pinned MediaPipe loader and WASM files, including the no-SIMD fallback.
- Initialize the MediaPipe loader inside the pose worker without depending on module-worker global behavior.
- Permit WebAssembly compilation through the strict application Content Security Policy.
- Close transferred camera frames even when inference throws.
- Exercise camera preview and actual pose inference in the Electron E2E flow.
- Correct the expected Linux artifact names used by release validation.

## Install

- **macOS:** Download the universal DMG or ZIP. Because this build is unsigned, use Finder's **Open** action if Gatekeeper blocks the first launch.
- **Windows:** Download the x64 EXE installer or portable ZIP. Because this build is unsigned, Microsoft Defender SmartScreen may show an unrecognized-app warning.
- **Linux:** Prefer AppImage for a portable launch, DEB for Ubuntu/Debian, RPM for Fedora-family distributions, or tar.gz for a manual install. You may need to mark the AppImage executable.

Compare downloads against `SHASUMS256.txt` before installing.

## Known limitations

- Apple Developer ID and Windows Authenticode signing are not configured.
- Automatic updates are disabled.
- Real-camera validation is complete on Apple Silicon macOS. Windows and Linux clean-machine camera checks remain part of the prerelease compatibility work.
- This software provides ergonomic reminders and is not a medical device.

If a camera remains unavailable, see the [camera troubleshooting guide](https://github.com/vivaanseth/Posture/blob/main/docs/troubleshooting-camera.md).
