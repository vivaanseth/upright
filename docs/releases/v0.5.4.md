# Posture v0.5.4

`v0.5.4` is the public beta release for the corrected `v0.5.x` line. It includes the `v0.5.2` reminder, camera-cleanup, calibration-cancellation, and local sound fixes, plus release-packaging corrections discovered during validation.

## Fixed

- Release publication no longer downloads Anchore's extra default `posture-build.cyclonedx.json` artifact; only the intended platform installers, platform SBOMs, and checksum manifest are published.
- Packaged apps no longer include the already-bundled runtime `node_modules` tree after moving runtime packages into production dependencies.
- Linux packages include desktop metadata so GNOME/KDE-style environments can better associate the installed launcher with running Posture windows.
- Reminder timing uses the main process's monotonic clock instead of accepting renderer-provided timestamps.
- Camera streams and pose workers are cleaned up if either half of startup fails.
- In-progress calibration is cancelled when backing out, leaving calibration views, hiding setup before tracking starts, pausing, stopping, resetting, or quitting.
- The reminder sound setting plays a subtle local-only cue when enabled.
- Adaptive sampling can recover toward higher FPS after sustained low latency and low dropped-frame rates.
- Media permission checks require explicit video-only requests.
- Missing `app://` assets return controlled 404 responses.

## Notes

- `v0.5.2` and `v0.5.3` were tagged but did not publish GitHub releases because release validation blocked incorrect artifacts.
- Camera processing remains local to the device.
- No camera frames, images, videos, or raw landmark sequences are saved by Posture.
- This beta release still requires the documented compatibility, performance, and accuracy evidence before a stable compatibility claim.
