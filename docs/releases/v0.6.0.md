# Upright 0.6.0

This release introduces the Upright name and repository while retaining upgrade compatibility with Posture 0.5.4.

## Upgrade notes

- Settings, calibrations, session summaries, camera permission state, and local Chromium data remain in the existing `posture-desktop` data directory.
- The application ID and Linux DEB/RPM package identity are unchanged, so supported package-manager upgrades retain the same installation record.
- On macOS, quit Posture before installing Upright. After confirming Upright has retained your data, remove the old `Posture.app` file because the two application filenames can coexist.
- Existing Posture exports remain valid historical records. New exports use Export V3 and identify the application as Upright.

## Highlights

- Upright product identity, launcher names, executables, artifacts, documentation, and repository links.
- Recent session history, explicit camera/recovery state, advanced memory-only diagnostics, and a test-reminder onboarding step.
- Stronger camera ownership, calibration compatibility, worker recovery, IPC isolation, and renderer failure handling.
- Complete runtime SBOM dependency relationships with a checksummed MediaPipe model component.

Upright remains beta software while the complete real-hardware compatibility, performance, accuracy, and privacy evidence matrix is being recorded.
