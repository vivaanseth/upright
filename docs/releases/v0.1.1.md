# Posture v0.1.1 camera hotfix

This prerelease fixes the onboarding defect that could prevent Posture from requesting camera access or listing a usable camera. It remains an unsigned early prototype and is not yet represented as a stable or fully validated cross-platform release.

## Fixed

- Request macOS camera permission only after the Privacy screen's Continue action.
- Refresh camera enumeration after access and merge the active stream into the picker.
- Recover from a stale saved camera identifier by asking for the default camera.
- Stop partial streams after setup failures and when navigating backward.
- Restrict media permission and renderer trust checks to exact application origins.
- Prevent camera selection from triggering unrelated login-item changes.

## Install

- **macOS:** Download the universal DMG or ZIP. Because this build is unsigned, use Finder's **Open** action if Gatekeeper blocks the first launch.
- **Windows:** Download the x64 EXE installer or portable ZIP. Because this build is unsigned, Microsoft Defender SmartScreen may show an unrecognized-app warning.
- **Linux:** Prefer AppImage for a portable launch, DEB for Ubuntu/Debian, RPM for Fedora-family distributions, or tar.gz for a manual install. You may need to mark the AppImage executable.

Compare downloads against `SHASUMS256.txt` before installing.

## Known limitations

- Apple Developer ID and Windows Authenticode signing are not configured.
- Automatic updates are disabled.
- Real-camera validation is complete on Apple Silicon macOS; Windows and Linux clean-machine camera checks remain part of the prerelease validation matrix.
- This software provides ergonomic reminders and is not a medical device.

If the camera remains unavailable, see the [camera troubleshooting guide](https://github.com/vivaanseth/Posture/blob/main/docs/troubleshooting-camera.md).
